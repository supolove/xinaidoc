支付异步通知文档

简要描述

支付完成后，支付中心会将用户支付完成的结果通知到CP,CP请按照文档规范返回应答数据.

注意:

  • 同样的通知可能会多次发送给CP,CP必须能够正确处理重复通知
  • 当CP未作出正确的应答,会进行通知重发.

请求方式

  • POST

通知参数

参数名 必选 数据类型 参数类型 说明
appId 是 string query 游戏ID
orderNo 是 string query 支付中心订单号
cpOrderId 是 string query cp订单号
state 是 string query 订单状态(-1:支付失败 2:支付成功)
productId 是 string query 商品ID
extension 是 string query cp扩展参数
money 是 int query 支付金额(分)
sign 是 string query 签名参数,参考签名过程进行校验
version 否 string query 接口版本
roleId 是 string query 角色ID
isTest 是 boolean query 是否测试渠道
serverId 是 string query 服务器ID
currency 是 string query 币种(CNY:人民币 暂时只支持人民币)
notifyId 是 long query 通知ID,用于订单合法性验证
timestamp 是 long query 时间戳(毫秒)

返回结果

CP确认完成通知无误,返回字符串ok即可,其他可返回对应的错误信息.

签名过程

  1. 将接收到的通知参加集合按照ASCII码从小到大排序,使用URL键值对的格式拼接成字符串

    注意null值不要拼接成字符串null

  2. 将上面的字符串最后拼接上&key=得到签名原文字符串，并对签名原文字符串进行MD5运算,再转换为大写

// appKey=IemGW5pM7FMP07O9n91vN1WD2CtDW5OU
示例参数:
    "appId" -> "2wsQ0C8J85uHX388qQ"
    "cpOrderId" -> "1552529678"
    "currency" -> "CNY"
    "notifyId" -> "2357940330784884173"
    "orderNo" -> "xa201903142357937260789237196"
    "productId" -> "testCpProduct1"
    "money" -> "0"
    "isTest" -> "true"
    "roleId" -> "10000000"
    "serverId" -> "testID"
    "state" -> "1"
    "timestamp" -> "1552621175751"
    "version" -> "1.0.1"
    "extension" -> null
步骤一:
appId=2wsQ0C8J85uHX388qQ&cpOrderId=1552529678&currency=CNY&isTest=true&money=0&notifyId=2357940330784884173&orderNo=xa201903142357937260789237196&productId=testCpProduct1&roleId=10000000&serverId=testID&state=1&timestamp=1555418686198&version=1.0.1
步骤二:
MD5(appId=2wsQ0C8J85uHX388qQ&cpOrderId=1552529678&currency=CNY&isTest=true&money=0&notifyId=2357940330784884173&orderNo=xa201903142357937260789237196&productId=testCpProduct1&roleId=10000000&serverId=testID&state=1&timestamp=1555418827159&version=1.0.1&key=IemGW5pM7FMP07O9n91vN1WD2CtDW5OU).toUpperCase()
