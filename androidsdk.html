Android聚合sdk接入文档

一、概要

1.1 目的

本文档主要针对Android版SDK的各个接口，制定出相应的接口说明及调用方法。开发人员可以通过阅读本文档并结合测试DEMO接入SDK，本SDK所有接口均需在主线程调用。

  • 游戏方提供的游戏包必须不能混淆和加固，否则会打包失败。

1.2 SDK流程图

用户登录时序图

用户充值时序图

二、导入资源

本SDK目前支持Android2.3及以上的系统版本，为兼容Android 4.0及以上版本手机，编译时请使用Android 4.2或以上版本

  • 注：导入demo工程请以utf-8编码。

2.1 导入jar包

将下载解压出来（资源jar）文件夹里面的jar包拷贝到libs目录下。

三、应用工程的AndroidManifest.xml配置

3.1 target sdk version

华为小米等渠道将拒绝target SDK version等级不到26的新应用上架 请检查该版本>=26

gradle配置：

defaultConfig {
    applicationId "com.xxx"
    minSdkVersion 22
    targetSdkVersion 26
    versionCode 100
    versionName "1.0"
}

四、SDK方法调用

4.1 Application实现

应用方需自行实现Application，需继承com.lib.xh.sdk.XHApplication 并配置到AndroidManifest.xml。

public class MyApplication extends XHApplication {

    @Override
    public void onCreate() {
        super.onCreate();
        //TODO 实现您的逻辑
    }
}

4.2 闪屏动画

应用方入口Activity需继承com.lib.xh.sdk.XHSplashActivity。 注意：游戏中相关操作，如：游戏初始化等，请勿在本activity进行操作。因为有些渠道是没有闪屏页的

闪屏Activity和游戏Activity在清单中必须添加如下配置，否则无法通过审核

闪屏Activity添加splash_activity属性 

<activity>
    <meta-data android:name="splash_activity" android:value="true"/>
</activity>

游戏Activity添加game_activity属性 

<activity>
    <meta-data android:name="game_activity" android:value="true"/>
</activity>

闪屏代码如下

public class SplashActivity extends XHSplashActivity {

    @Override
    protected void onSplashBefore(Context context, Bundle savedInstanceState) {
        // TODO 闪屏动画播放前
    }
    @Override
    protected void onSplashFinish() {
        // TODO 闪屏动画播放完毕
        startActivity(new Intent(SplashActivity.this,GameActivity.class));
        finish();
    }
}

4.3 初始化接口

4.3.1 应用初始化

请在Application.onCreate中调用以下初始化应用方法：

  • 测试环境与生产环境有不同的appId与appKey，切换环境时请注意检查AndroidManifest.xml的appId与appKey是否为对应的值
参数 类型 描述
isLandscape boolean 游戏屏幕方向 横屏：true 竖屏：false
isDebug boolean 测试于生产环境切换
(注意接入对应服务端)
测试：true 生产：false
游戏上线时必须修改为：false		 
XHConfig config = new XHConfig();
config.setLandscape(true);//设置方向 横屏 true 竖屏false
config.setDebug(true);//设置debug模式,对应测试环境，正式打包时，请修改为false
XHSDK.getInstance().initApplication(this,config);

4.3.2 Game初始化

请在游戏Activity,如:GameActivity中调用一下游戏初始化方法，并保证游戏运行期间 该Activity不被销毁，否则会引起crash。 渠道操作拉起游戏的情景下，有可能会出现游戏Activity被拉起两个的情况, 所以在游戏Activity的onCreate里面需要检测当前Activity是否是重复的游戏Activity，如重复请去除。 注意：该方法必须要先于sdk.onCreate()方法执行

void initGameActivity(Activity activity,String versionPack);
参数 类型 描述
activity Activity 游戏Activity
versionPack String 游戏自定义的版号，请正确传入游戏版本号，以便数据统计正确

4.3.3 统一的回调接口

请在游戏的Activity的onCreate方法中使用该代码初始化监听器
该监听器包括所有sdk的回调，接入人员需要通过回调处理好游戏的逻辑
回调方法包括

  1. 初始化回调
  2. 登录回调
  3. 登出回调
  4. 切换账号回调
  5. 充值回调
  6. 退出程序回调

示例代码：

sdk.setStateCallBack(initStateCallback());

private XHStateCallBack initStateCallback(){
    return new XHStateCallBack() {

        @Override
        public void onInitSuccess() {
            Log.d(TAG, "onInitSuccess: ");
        }

        @Override
        public void onInitFail(XHErrorInfo xhErrorInfo) {
            Log.d(TAG, "onInitFail: ");
        }

        @Override
        public void onLoginSuccess(XHGotUserInfo userInfo) {
            Log.d(TAG, "onLoginSuccess: ");
            //进入游戏
            toGameContent();
            // sdk返回的用户ID 支付时需传入
            userId = userInfo.getUid() + "";
//            // 校验用户有效性需使用
            String secretKey = userInfo.getSecretKey();
//            toast("登录成功\n" + accessToken);
            textView.setText("游戏页面\nuserId:" + userId);
            //提交游戏用户信息
            subUserInfo();
        }

        @Override
        public void onLoginFail(XHErrorInfo xhErrorInfo) {
            Log.d(TAG, "onLoginFail: " + xhErrorInfo.getErrorMsg());
            switch (xhErrorInfo.getErrorCode()) {
                case XHError.CODE_LOGIN_CANCEL:
                    // 登陆取消
                    break;
                case XHError.CODE_LOGIN_LOADING:
                    // 正在登陆
                    break;
                case XHError.CODE_USER_INVALID:
                    //登陆失败XHError.CODE_USER_INVALID返回,请厂商在收到此返回时把游戏退出到首页，否则测试不会通过，谢谢合作！
                    returnGameLogin();
                    break;
                default:
                    // 其他异常
                    break;
            }
            toast(xhErrorInfo.getErrorMsg());
        }

        @Override
        public void onLogoutSuccess() {
            Log.d(TAG, "onLogoutSuccess: ");
            returnGameLogin();
        }

        @Override
        public void onLogoutFail(String errorMessage) {
            Log.d(TAG, "onLogoutFail: " + errorMessage);
        }

        @Override
        public void onPaySuccess(String orderId) {
            // 此时CP可记录订单编号，提交给后台服务器，等待到账通知，如果到账即可发货
            toast("已发起支付，请查询服务器是否到账" + orderId);
            Log.d(TAG, "onPaySuccess: ");
        }

        @Override
        public void onPayFail(XHErrorInfo xhErrorInfo) {
            Log.d(TAG, "onPayFail: " );
            switch (xhErrorInfo.getErrorCode()) {
                case XHError.CODE_WITHOUT_LOGIN:
                    // 尚未登陆
                    break;
                case XHError.CODE_NEED_LOGIN_AGAIN:
                    // 需要重新登陆
                    break;
                case XHError.CODE_PAY_CANCEL:
                    // 支付取消
                    break;
                case XHError.CODE_IN_PAYMENT:
                    // 正在付款
                    break;
                default:
                    // 其他异常
                    break;
            }
            toast(xhErrorInfo.getErrorMsg());
        }

        @Override
        public void onQuit() {
            Log.d(TAG, "onQuit: ");
            // 回收sdk资源
            sdk.destroySDK(GameActivity.this);
            GameActivity.this.finish();
            System.exit(0);
        }

        @Override
        public void onSwitchAccountSuccess(XHGotUserInfo userInfo) {
            Log.d(TAG, "onSwitchAccountSuccess: ");
            // 切换登录成功
            // 进入游戏
            toGameContent();
            userId = userInfo.getUid() + "";
            userName = userInfo.getUsername();
        }

        @Override
        public void onSwitchAccountFail(String erroe) {
            Log.d(TAG, "onSwitchAccountFail: ");
            // 请厂商在收到此返回时把游戏退出到首页，否则测试不会通过，谢谢合作！
            // 返回登录界面
            returnGameLogin();
        }
    };
}

注意

  1. onSwitchAccountSuccess返回登录成功的用户信息，游戏切换账号后，需要使用该返回的用户信息进入服务器，否则切换账号不成功，不能通过审核。
  2. 在接收到onFail方法请退出当前游戏用户，例如回到登录界面重新登录

4.4 登陆/注册接口

4.4.1 接口一

void login(Context context);

通过实现XHCallBack<XHGotUserInfo>接口获取登陆结果通知，登陆成功将回调onSuccess可获取用户信息，如登陆异常将回调相应的状态吗和状态消息。 (登陆失败XHError.CODE_USER_INVALID返回,请厂商在收到此返回时把游戏退出到登录场景，否则测试不会通过，谢谢合作！) 登陆成功后需验证用户有效性，厂商需接入验证接口，详见：SDK服务器接口。

Code Msg
XHError.CODE_LOGIN_CANCEL 用户取消登陆操作
XHError.CODE_LOGIN_LOADING 用户正在登陆
XHError.CODE_USER_INVALID 用户失效

可参考以下代码实现SDK的注册/登陆调用处理:

sdk.login(GameActivity.this);

需要测试登陆功能请使用下列测试账号：

4.4.2 接口二

void login2(Context context, XHPlatform platform);

本接口兼用单渠道多账户系统登陆，如应用宝：支持QQ或微信登陆，其余逻辑均与接口一相同。 为了使游戏自定义微信，qq登录按钮布局，使更兼容游戏场景，提供了区分qq、微信登录入口。

参数 描述
XHPlatform.QQ QQ登陆
XHPlatform.WX 微信登陆

4.5.2 支付调用

void doPayBySDK(Context context, XHCpPayInfo cpPayInfo);

通过实现XHCallBack<Bundle>接口获取支付结果通知，onSuccess的回调仅表示用户是否完成支付操作，由于支付到账延迟的情况偶有发生，支付是否成功将通过后台通知开发商的支付回调地址。只有开发商的服务器接到成功的通知，支付才算成功。

4.6 退出接口

void quit(Context context)

在调用此接口前需要判断渠道是否存在退出接口，可通过boolean hadPlatformQuitUI()方法判断，true表示存在渠道退出，false表示不存在渠道退出 如果存在再调用该接口，否则游戏自行实现退出游戏操作，调用该接口，点击确定时将回调quit（）方法，取消时回调cancel（）方法；

4.7 销毁接口

void destroySDK(Context context)

此接口将回收SDK的资源，在游戏退出时需要调用销毁接口，以便释放内存。
